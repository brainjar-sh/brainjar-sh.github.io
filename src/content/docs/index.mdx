## How it works

Instead of one monolithic config file, brainjar separates agent behavior into composable layers. Each layer is a markdown file. Mix and match them per project, per task, or per session.

<CardGrid>
	<Card title="Soul" icon="heart">
		Who the agent **is** — voice, character, standards. The constant across all tasks. Think personality.
	</Card>
	<Card title="Persona" icon="puzzle">
		How the agent **works** — role and workflow. An engineer works differently than a reviewer. Switch per task.
	</Card>
	<Card title="Rules" icon="warning">
		What the agent **must follow** — behavioral constraints and guardrails. Apply regardless of persona.
	</Card>
	<Card title="Brain" icon="rocket">
		A saved snapshot of soul + persona + rules. Activate a full configuration in one command.
	</Card>
</CardGrid>

## Quick install

```bash
bun install -g @brainjar/cli
brainjar init --default
```

That gives you a soul, a persona, and rules — all wired into your `CLAUDE.md` and ready to go.

## Agent-friendly by design

brainjar is built with [incur](https://github.com/wevm/incur), a CLI framework designed for agent consumption. Every command auto-detects whether it's talking to a human or an agent:

- **Agents** get structured output (TOON, JSON, YAML) with typed schemas
- **Humans** get formatted text with helpful context and next steps
- **Discovery** via `--llms`, MCP server, or skill files — agents can learn the full CLI without documentation

```bash
# Human output
brainjar status

# Agent output (auto-detected when piped, or explicit)
brainjar status --json
brainjar status --format yaml
```

## AI-friendly docs

Point your agent at [`brainjar.sh/llms.txt`](https://brainjar.sh/llms.txt) for a doc index, or append `.md` to any page URL for raw markdown (e.g. [`brainjar.sh/getting-started.md`](https://brainjar.sh/getting-started.md)).
