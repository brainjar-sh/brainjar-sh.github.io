## How it works

Instead of one monolithic config file, brainjar separates agent behavior into composable layers. Each layer is a markdown file. Mix and match them per project, per task, or per session.

	<LinkCard title="👻 Soul" href="/concepts/souls/" description="Who the agent is — voice, character, standards. The constant across all tasks." />
	<LinkCard title="🎭 Persona" href="/concepts/personas/" description="How the agent works — role and workflow. Switch per task." />
	<LinkCard title="🚧 Rules" href="/concepts/rules/" description="What the agent must follow — behavioral constraints and guardrails." />

## Quick install

```bash
bun install -g @brainjar/cli
brainjar init --default
```

That gives you a soul, a persona, and rules — all wired into your `CLAUDE.md` and ready to go.

## Built for agents

brainjar is built with [incur](https://github.com/wevm/incur), a CLI framework designed for agent consumption. Every command auto-detects whether it's talking to a human or an agent — structured output (TOON, JSON, YAML) when piped, formatted text with context when interactive. Agents discover the full CLI via `--llms`, MCP server, or skill files.

```bash
brainjar status                     # human: formatted with context
brainjar status --format toon       # agent: structured data
```

These docs are agent-readable too. Point your agent at [`brainjar.sh/llms.txt`](https://brainjar.sh/llms.txt) for a doc index, or append `.md` to any page URL for raw markdown (e.g. [`/getting-started.md`](https://brainjar.sh/getting-started.md)).
